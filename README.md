# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

### Интерфейсы данных

#### IProduct

Интерфейс описывает данные товара, доступного для покупки в интернет-магазине.

**Поля:**

- `id: string` - уникальный идентификатор товара
- `description: string` - описание товара
- `image: string` - путь к изображению товара
- `title: string` - название товара
- `category: string` - категория товара
- `price: number | null` - цена товара в синапсах. Может быть `null`, если товар недоступен для покупки. Товары без цены нельзя добавить в корзину, кнопка в карточке preview будет заблокирована и отобразит текст "Недоступно"

#### IBuyer

Интерфейс описывает данные покупателя, необходимые для оформления заказа.

**Поля:**

- `payment: TPayment` - способ оплаты заказа
- `email: string` - электронная почта покупателя
- `phone: string` - номер телефона покупателя
- `address: string` - адрес доставки

## Модели данных

Модели данных отвечают за хранение и работу с данными приложения. Каждая модель отвечает за свою зону ответственности и реализует методы для управления данными в рамках своей области.

### Класс Catalog

Класс предназначен для хранения данных о товарах и управления ими.

**Назначение:**  
Хранение массива всех товаров каталога и текущего выбранного товара для подробного отображения.

**Конструктор:**  
Конструктор не принимает параметров. Инициализирует пустой массив товаров и значение `null` для выбранного товара.

**Поля класса:**

- `items: IProduct[]` - массив товаров, доступных в каталоге
- `preview: IProduct | null` - товар, выбранный для подробного отображения. Может быть `null`, если товар не выбран

**Методы класса:**

- `setItems(items: IProduct[]): void` - сохраняет массив товаров, полученный в параметре метода, в поле `items`
- `getItems(): IProduct[]` - возвращает массив всех товаров из модели
- `getItemById(id: string): IProduct | null` - возвращает товар по его `id`, переданному в параметре. Возвращает `null`, если товар с указанным `id` не найден
- `setPreview(item: IProduct | null): void` - сохраняет товар, переданный в параметре, для подробного отображения в поле `preview`. Может принимать `null` для очистки выбранного товара
- `getPreview(): IProduct | null` - возвращает товар, выбранный для подробного отображения. Возвращает `null`, если товар не выбран

### Класс Cart

Класс предназначен для хранения товаров, выбранных пользователем для покупки.

**Назначение:**  
Управление списком товаров, добавленных пользователем в корзину.

**Конструктор:**  
Конструктор не принимает параметров. Инициализирует пустой массив товаров.

**Поля класса:**

- `items: IProduct[]` - массив товаров, добавленных в корзину

**Методы класса:**

- `getItems(): IProduct[]` - возвращает массив всех товаров, находящихся в корзине
- `addItem(item: IProduct): void` - добавляет товар, переданный в параметре, в массив корзины
- `removeItem(item: IProduct): void` - удаляет товар, переданный в параметре, из массива корзины
- `clear(): void` - очищает корзину, удаляя все товары из массива
- `getTotal(): number` - возвращает общую стоимость всех товаров в корзине (сумма цен всех товаров). Если у товара цена `null`, такой товар не учитывается в общей стоимости
- `getCount(): number` - возвращает количество товаров в корзине
- `contains(id: string): boolean` - проверяет наличие товара в корзине по его `id`, переданному в параметре. Возвращает `true`, если товар найден, и `false` в противном случае

### Класс Buyer

Класс предназначен для хранения данных покупателя, необходимых для оформления заказа.

**Назначение:**  
Хранение и валидация данных покупателя при оформлении заказа.

**Конструктор:**  
Конструктор не принимает параметров. Инициализирует все поля класса значениями по умолчанию (пустые строки или `null` в зависимости от типа поля).

**Поля класса:**

- `payment: TPayment | null` - способ оплаты заказа. Может быть `null`, если способ оплаты не выбран
- `address: string` - адрес доставки
- `phone: string` - номер телефона покупателя
- `email: string` - электронная почта покупателя

**Методы класса:**

- `setData(data: Partial<IBuyer>): void` - сохраняет данные покупателя в модели. Принимает объект с частичными данными покупателя, что позволяет обновить одно или несколько полей, не затрагивая остальные. Если поле отсутствует в переданном объекте, его значение в модели остается неизменным
- `setPayment(payment: TPayment): void` - сохраняет способ оплаты
- `setAddress(address: string): void` - сохраняет адрес доставки
- `setPhone(phone: string): void` - сохраняет номер телефона
- `setEmail(email: string): void` - сохраняет электронную почту
- `getData(): IBuyer` - возвращает объект со всеми данными покупателя
- `clear(): void` - очищает все данные покупателя, сбрасывая значения всех полей в значения по умолчанию
- `validate(): Partial<Record<keyof IBuyer, string>>` - валидирует данные покупателя. Возвращает объект, ключами которого являются названия полей, а значениями - тексты ошибок валидации для соответствующих полей. Если поле валидно (не пустое), оно отсутствует в возвращаемом объекте. Пустой объект означает, что все поля валидны. Правила валидации: каждое поле должно быть не пустым (не пустая строка для строковых полей, не `null` для поля `payment`)

## Слой коммуникации

### Класс ShopAPI

Класс представитель коммуникационного слоя, отвечает за получение данных с сервера и отправку данных на сервер.

**Назначение:**  
Взаимодействие с API сервера через методы класса Api.

**Конструктор:**  
`constructor(api: IApi)` - принимает объект, соответствующий интерфейсу IApi, который будет использован для выполнения запросов на сервер.

**Поля класса:**

- `api: IApi` - объект для выполнения запросов на сервер

**Методы класса:**

- `getProductList(): Promise<IProductListResponse>` - выполняет GET запрос на эндпоинт `/product/` и возвращает промис с объектом, содержащим массив товаров
- `createOrder(order: IOrderRequest): Promise<IOrderResponse>` - выполняет POST запрос на эндпоинт `/order` и передает в него данные заказа, полученные в параметрах метода. Возвращает промис с результатом создания заказа

## Слой представления

Компоненты представления отвечают за отображение данных на странице и взаимодействие с пользователем. Каждый компонент наследуется от базового класса `Component` и реализует методы для работы с DOM-элементами.

### Базовые классы компонентов

#### Card

Базовый класс для всех компонентов карточек товаров. Наследуется от `Component`.

**Назначение:**  
Предоставляет общую функциональность для отображения названия и цены товара.

**Конструктор:**  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент карточки.

**Поля класса:**

- `titleElement: HTMLElement | null` - элемент для отображения названия товара
- `priceElement: HTMLElement | null` - элемент для отображения цены товара

**Сеттеры:**

- `title: string` - устанавливает название товара
- `price: number | null` - устанавливает цену товара. Если цена равна `null`, отображается текст "Бесценно"

#### Form

Абстрактный базовый класс для всех компонентов форм. Наследуется от `Component`.

**Назначение:**  
Предоставляет общую функциональность для работы с формами: обработка отправки формы, отображение ошибок валидации, управление активностью кнопки отправки.

**Конструктор:**  
`constructor(events: IEvents, container: HTMLElement)` - принимает объект для работы с событиями и ссылку на DOM элемент формы.

**Поля класса:**

- `submitButton: HTMLButtonElement` - кнопка отправки формы
- `errorsElement: HTMLElement` - элемент для отображения ошибок валидации

**Абстрактные методы:**

- `getSubmitEventName(): string` - должен возвращать название события, которое будет сгенерировано при отправке формы
- `getErrorMessages(errors: Partial<IBuyerErrors>): string[]` - должен возвращать массив сообщений об ошибках для отображения

**Сеттеры:**

- `errors: Partial<IBuyerErrors>` - устанавливает ошибки валидации и отображает их в форме
- `valid: boolean` - управляет активностью кнопки отправки формы

### Компоненты карточек товаров

#### CardCatalog

Компонент для отображения карточки товара в каталоге. Наследуется от `Card`.

**Назначение:**  
Отображение товара в галерее каталога с возможностью открытия детальной информации по клику.

**Конструктор:**  
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает ссылку на DOM элемент и опциональный объект с обработчиками событий.

**Поля класса:**

- `imageElement: HTMLImageElement | null` - элемент изображения товара
- `categoryElement: HTMLElement | null` - элемент категории товара

**Сеттеры:**

- `image: string` - устанавливает изображение товара
- `category: string` - устанавливает категорию товара и соответствующий CSS класс

**События:**

- Генерирует событие `card:select` при клике на карточку (через обработчик `actions.onClick`)

#### CardBasket

Компонент для отображения товара в корзине. Наследуется от `Card`.

**Назначение:**  
Отображение товара в списке корзины с возможностью удаления.

**Конструктор:**  
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает ссылку на DOM элемент и опциональный объект с обработчиками событий.

**Поля класса:**

- `indexElement: HTMLElement` - элемент для отображения порядкового номера товара
- `buttonElement: HTMLButtonElement` - кнопка удаления товара из корзины

**Сеттеры:**

- `index: number` - устанавливает порядковый номер товара в корзине

**События:**

- Генерирует событие `card:remove` при клике на кнопку удаления (через обработчик `actions.onClick`)

#### CardPreview

Компонент для отображения детальной информации о товаре в модальном окне. Наследуется от `Card`.

**Назначение:**  
Отображение полной информации о товаре с возможностью добавления/удаления из корзины.

**Конструктор:**  
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает ссылку на DOM элемент и опциональный объект с обработчиками событий.

**Поля класса:**

- `imageElement: HTMLImageElement | null` - элемент изображения товара
- `categoryElement: HTMLElement | null` - элемент категории товара
- `textElement: HTMLElement` - элемент для отображения описания товара
- `buttonElement: HTMLButtonElement` - кнопка добавления/удаления товара из корзины

**Сеттеры:**

- `image: string` - устанавливает изображение товара
- `category: string` - устанавливает категорию товара и соответствующий CSS класс
- `description: string` - устанавливает описание товара
- `buttonText: string` - устанавливает текст кнопки ("Купить" или "Удалить из корзины")
- `buttonDisabled: boolean` - блокирует/разблокирует кнопку. Если у товара цена равна `null`, кнопка блокируется и отображает текст "Недоступно"

**События:**

- Генерирует событие `card:add` при клике на кнопку "Купить" (через обработчик `actions.onClick`)
- Генерирует событие `card:remove` при клике на кнопку "Удалить из корзины" (через обработчик `actions.onClick`)

**Особенности:**

- Если у товара цена равна `null`, кнопка блокируется и отображает текст "Недоступно"
- Товары без цены нельзя добавить в корзину

#### Modal

Компонент модального окна.

**Назначение:**  
Отображение различного контента в модальном окне (карточка товара, корзина, формы заказа и т.д.).

**Конструктор:**  
`constructor(events: IEvents, container: HTMLElement)` - принимает объект для работы с событиями и ссылку на DOM элемент модального окна.

**Поля класса:**

- `containerElement: HTMLElement` - контейнер модального окна
- `closeButton: HTMLButtonElement` - кнопка закрытия модального окна
- `contentElement: HTMLElement` - элемент для отображения контента

**Методы класса:**

- `open(): void` - открывает модальное окно (добавляет класс `modal_active`)
- `close(): void` - закрывает модальное окно (удаляет класс `modal_active`) и генерирует событие `modal:close`
- `hasContentType(type: ModalContentType): boolean` - проверяет наличие контента определенного типа в модальном окне

**Сеттеры:**

- `content: HTMLElement | null` - устанавливает контент модального окна. Если передано `null`, контент очищается

**Геттеры:**

- `content: HTMLElement | null` - возвращает текущий контент модального окна

**События:**

- Генерирует событие `modal:close` при закрытии модального окна (клик на кнопку закрытия или клик вне модального окна)

#### Success

Компонент окна успешного оформления заказа.

**Назначение:**  
Отображение сообщения об успешном оформлении заказа с информацией о списанной сумме.

**Конструктор:**  
`constructor(events: IEvents, container: HTMLElement)` - принимает объект для работы с событиями и ссылку на DOM элемент.

**Поля класса:**

- `descriptionElement: HTMLElement` - элемент для отображения сообщения об успешном оформлении заказа

**Сеттеры:**

- `total: number` - устанавливает сумму списанных синапсов в сообщении

**События:**

- Генерирует событие `success:close` при клике на кнопку закрытия

## Презентер

Презентер является связующим звеном между слоями Model и View. Он содержит основную бизнес-логику приложения и отвечает за обработку событий, генерируемых моделями данных и представлениями.

**Назначение:**  
Обработка событий от моделей данных и представлений, координация взаимодействия между слоями приложения, управление состоянием приложения.

**Реализация:**  
В данном проекте презентер реализован в основном файле приложения `main.ts`. Так как приложение имеет только одну страницу, достаточно одного презентера, который отвечает за логику работы этой страницы.

**Принципы работы:**

- Презентер подписывается на события от моделей данных и представлений
- При получении события от модели данных презентер обновляет представление
- При получении события от представления презентер обновляет модель данных
- Презентер не генерирует события, а только обрабатывает их
- Представление перерисовывается только при изменении данных в модели или при открытии модального окна

**Вспомогательные функции:**

- `validateOrderForm(buyer: Buyer)` - функция валидации формы заказа. Проверяет только поля `payment` (способ оплаты) и `address` (адрес доставки). Возвращает объект с ошибками валидации и флагом валидности. Используется для определения активности кнопки "Далее" в форме заказа - кнопка активна только если выбрана оплата и заполнен адрес доставки.

**Особенности реализации:**

- **Обработка товаров без цены:** Если у товара цена равна `null`, кнопка в карточке preview блокируется (`buttonDisabled = true`) и отображает текст "Недоступно". Такие товары нельзя добавить в корзину.
- **Валидация формы заказа:** Кнопка "Далее" в форме заказа активна только при выполнении всех условий: выбран способ оплаты, заполнено поле адреса доставки и отсутствуют ошибки валидации для этих полей. Email и телефон проверяются на следующем шаге оформления заказа.

## События приложения

Приложение использует событийно-ориентированный подход для взаимодействия между компонентами. Все события генерируются моделями данных или компонентами представления и обрабатываются презентером в файле `main.ts`.

### События от моделей данных

Модели данных генерируют события при изменении своего состояния, уведомляя презентер о необходимости обновления представления.

#### `items:changed` (Catalog)

**Источник:** `Catalog`  
**Когда генерируется:** При изменении массива товаров в каталоге (метод `setItems`)  
**Данные:** `{ items: IProduct[] }` - массив всех товаров каталога  
**Обработка:** Презентер обновляет галерею товаров, создавая карточки для каждого товара из каталога

#### `preview:changed` (Catalog)

**Источник:** `Catalog`  
**Когда генерируется:** При изменении выбранного товара для подробного просмотра (метод `setPreview`)  
**Данные:** `{ preview: IProduct | null }` - товар для просмотра или `null`, если товар не выбран  
**Обработка:** Презентер открывает модальное окно с детальной информацией о товаре или закрывает его, если `preview` равен `null`

#### `items:changed` (Cart)

**Источник:** `Cart`  
**Когда генерируется:** При добавлении товара (метод `addItem`), удалении товара (метод `removeItem`) или очистке корзины (метод `clear`)  
**Данные:** `{ items: IProduct[] }` - массив всех товаров в корзине  
**Обработка:** Презентер обновляет счетчик товаров в хедере, обновляет содержимое корзины и обновляет карточку preview, если она открыта

#### `data:changed` (Buyer)

**Источник:** `Buyer`  
**Когда генерируется:** При изменении любых данных покупателя (методы `setData`, `setPayment`, `setAddress`, `setPhone`, `setEmail`, `clear`)  
**Данные:** `{ data: IBuyer }` - объект со всеми данными покупателя  
**Обработка:** Презентер обновляет формы заказа и контактов, отображает ошибки валидации и управляет активностью кнопок отправки форм

### События от представлений

Компоненты представления генерируют события при взаимодействии пользователя с интерфейсом, уведомляя презентер о необходимости обновления моделей данных или выполнения действий.

#### `card:select`

**Источник:** `CardCatalog`  
**Когда генерируется:** При клике на карточку товара в каталоге  
**Данные:** `{ id: string }` - идентификатор выбранного товара  
**Обработка:** Презентер находит товар по ID в каталоге и устанавливает его как товар для просмотра (`catalogModel.setPreview`)

#### `card:add`

**Источник:** `CardPreview`  
**Когда генерируется:** При нажатии кнопки "Купить" в модальном окне с детальной информацией о товаре  
**Данные:** `{ id: string }` - идентификатор товара для добавления в корзину  
**Обработка:** Презентер находит товар по ID в каталоге и добавляет его в корзину (`cartModel.addItem`)

#### `card:remove`

**Источник:** `CardPreview`, `CardBasket`  
**Когда генерируется:** При нажатии кнопки "Удалить из корзины" в модальном окне с детальной информацией о товаре или кнопки удаления в корзине  
**Данные:** `{ id: string }` - идентификатор товара для удаления из корзины  
**Обработка:** Презентер находит товар по ID в каталоге и удаляет его из корзины (`cartModel.removeItem`)

#### `basket:open`

**Источник:** `Header`  
**Когда генерируется:** При нажатии на кнопку корзины в хедере  
**Данные:** Отсутствуют  
**Обработка:** Презентер открывает модальное окно с содержимым корзины

#### `basket:order`

**Источник:** `Basket`  
**Когда генерируется:** При нажатии кнопки "Оформить" в корзине  
**Данные:** Отсутствуют  
**Обработка:** Презентер открывает модальное окно с формой заказа (первый шаг оформления)

#### `order:next`

**Источник:** `Order` (через базовый класс `Form`)  
**Когда генерируется:** При отправке формы заказа (нажатие кнопки "Далее")  
**Данные:** Отсутствуют  
**Обработка:** Презентер проверяет валидность данных формы заказа (способ оплаты и адрес доставки). Если данные валидны, открывает модальное окно с формой контактов (второй шаг оформления)

#### `contacts:submit`

**Источник:** `Contacts` (через базовый класс `Form`)  
**Когда генерируется:** При отправке формы контактов (нажатие кнопки "Оплатить")  
**Данные:** Отсутствуют  
**Обработка:** Презентер отправляет заказ на сервер через API, очищает корзину и данные покупателя, открывает модальное окно с сообщением об успешном оформлении заказа

#### `order:payment:change`

**Источник:** `Order`  
**Когда генерируется:** При выборе способа оплаты (клик на кнопку "Онлайн" или "При получении")  
**Данные:** `{ payment: "online" | "offline" }` - выбранный способ оплаты  
**Обработка:** Презентер сохраняет способ оплаты в модели покупателя (`buyerModel.setPayment`)

#### `order:address:change`

**Источник:** `Order`  
**Когда генерируется:** При вводе текста в поле адреса доставки  
**Данные:** `{ address: string }` - введенный адрес доставки  
**Обработка:** Презентер сохраняет адрес доставки в модели покупателя (`buyerModel.setAddress`)

#### `contacts:email:change`

**Источник:** `Contacts`  
**Когда генерируется:** При вводе текста в поле email  
**Данные:** `{ email: string }` - введенный email  
**Обработка:** Презентер сохраняет email в модели покупателя (`buyerModel.setEmail`)

#### `contacts:phone:change`

**Источник:** `Contacts`  
**Когда генерируется:** При вводе текста в поле телефона  
**Данные:** `{ phone: string }` - введенный телефон  
**Обработка:** Презентер сохраняет телефон в модели покупателя (`buyerModel.setPhone`)

#### `modal:close`

**Источник:** `Modal`  
**Когда генерируется:** При закрытии модального окна (клик на кнопку закрытия или клик вне модального окна)  
**Данные:** Отсутствуют  
**Обработка:** Презентер сбрасывает выбранный товар для просмотра (`catalogModel.setPreview(null)`)

#### `success:close`

**Источник:** `Success`  
**Когда генерируется:** При нажатии кнопки закрытия в окне успешного оформления заказа  
**Данные:** Отсутствуют  
**Обработка:** Презентер закрывает модальное окно
